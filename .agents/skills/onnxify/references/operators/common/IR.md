# IR

Shared reference for ONNX operator docs generated by `src/Onnxify.AgentSkillGenerator`.

## Optional Arguments

- ONNX represents a missing optional input or output by using an empty string in that argument slot.
- Trailing optional arguments may be omitted entirely when no later arguments are present.
- When reading or emitting nodes, preserve argument order so optional positions keep their intended meaning.

## Why It Matters Here

- Recurrent operators and a few utility operators rely on optional inputs and outputs.
- Generated docs mention this rule so the skill can explain why a node may have fewer connected values than the full schema suggests.
