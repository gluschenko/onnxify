# Broadcasting

Shared reference for ONNX operator docs generated by `src/Onnxify.AgentSkillGenerator`.

## Rules

- Multidirectional broadcasting follows NumPy-style shape expansion from trailing dimensions.
- A dimension is compatible when the values are equal, or one of them is `1`.
- Missing leading dimensions behave as if the shorter shape were padded with leading `1` values.

## Unidirectional Case

- Unidirectional broadcasting means the secondary tensor must be broadcastable to the primary tensor's shape.
- In practice, this is the usual "parameter tensor expands to activation tensor" case used by operators such as `Gemm`, `PRelu`, and normalization layers.
